1. What is MapObjects—Java Edition?

MapObjects—Java is a suite of Java-based developer components for creating client- or server-side mapping and GIS applications. MapObjects—Java contains a collection of Java Archive (JAR) files that can be used to develop custom Java-based solutions. Version 2.0 of MapObjects—Java will also begin to extend into the server side of the client-server model, featuring the notion of a new kind of server-side layout component and map component. The core libraries provide a rich resource of GIS and mapping components built completely on the Java 2 platform. They can be used to integrate GIS and mapping into existing Java systems, for instance, or to leverage your ArcIMS services through thick clients. These MapObjects—Java 2.0 server-side objects have the ability to live within servlet containers, as well as the ability to be wrapped into Enterprise Java Beans (EJBs) that operate with the J2EE container.

2. Can I configure the MapObjects—Java JAR files to be referenced in a Java-based IDE? If so, which Java IDEs are recommended?

Yes. MapObjects—Java contains a suite of visual JavaBeans that can be added to custom design-time toolbars in IDEs, such as JBuilder and Sun ONE Studio. These beans can be used at design time in a drag-and-drop environment for building graphical user interfaces. JBuilder and Sun ONE Studio are good IDEs within which MapObjects—Java API works well. Many others may work as well but have not been fully tested.

3. Where can I find more information about support for MapObjects—Java?

The ESRI Online Support Center at http://support.esri.com provides a wealth of information. System requirements, samples, and any other pertinent software information can be found there. There are also details and procedures on the various support services offered for MapObjects—Java.

4. Can the components in MapObjects—Java run in the Java 2 Enterprise Edition (J2EE) environment?

Yes. Starting at version 2.0, the components in MapObjects—Java are architected to work in both J2SE and J2EE environments.

5. Does MapObjects—Java work with the geodatabase?

Enterprise geodatabases are exposed through ArcSDE. MapObjects supports read-only access to the default version of these geodatabases through ArcSDE. However, MapObjects can only see geometry as simple features and attributes in a geodatabase stored in ArcSDE. MapObjects does not respect custom features, relationship classes, feature-linked annotation, and so forth.  

6. Can I access/update ArcSDE data with MapObjects—Java API?

ArcSDE data can be accessed via the MapObjects—Java API, but there are no low level MapObects—Java API components to update the data.  

The ArcSDE Java API must be used to update data. The ArcSDE Java API can be imported into a MapObjects—Java application to perform these updates, but remember it is not the MapObjects—Java API that's doing this.

Furthermore, there is a recommended procedure to mix the ArcSDE Java API with a MapObjects—Java based application. First, connect to the ArcSDE database using the MapObjects—Java Content API. This is done by using the ArcSdeConnection within the esri.mo2.client.sde package. After retrieving the Source object, cast it to com.esri.mo2.client.sde.ArcSdeSource. From this object, use the getSeConnection method to obtain a low-level ArcSDE Java connection object to use. By doing it this way, both APIs can use a common SeConnection, thus keeping down the number of SDE licenses used to just one.
